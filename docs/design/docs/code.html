Multi-language pipeline

Configure typeDiagram as the single source of truth for DTOs in a polyglot project. This guide uses a TypeScript frontend + Rust backend, but the pattern works for any combination of languages.

Project layout

folder_open Directory Structure 
repo/
├── schemas/
│   ├── user.diag
│   └── auth.diag
├── frontend/ // TypeScript
│   └── src/types/generated/
└── backend/ // Rust
    └── src/models/generated/

Generate code for each target

Use the type-diag CLI to compile your schemas into language-specific DTOs. Each generator handles language idiomatic patterns like Option<T> in Rust or interfaces in TypeScript.

TypeScript Frontend

.ts
$ type-diag gen -l ts -i ./schemas -o ./frontend/src/types
Output: user.generated.ts, auth.generated.ts

Rust Backend

.rs
$ type-diag gen -l rust -i ./schemas -o ./backend/src/models
Output: user.rs, auth.rs
hub

Automated CI Sync

Integrate the pipeline into your GitHub Actions to ensure frontend and backend types are always in sync. If a schema change breaks a consumer, the build fails before merging.
